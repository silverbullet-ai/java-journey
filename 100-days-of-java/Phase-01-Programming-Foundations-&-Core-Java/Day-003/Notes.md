# Day 3 – Pseudocode

## What is Pseudocode?

**Pseudocode** is an informal way of writing algorithms using simple, human-readable language that resembles programming structure but is not bound to any specific programming language.

It helps in designing logic before writing actual code.

---

## Key Characteristics of Pseudocode

- **Plain Language** → Easy to read and understand  
- **Algorithm Focused** → Focuses on logic, not syntax  
- **Language-Agnostic** → Not tied to any programming language  

---

## Purpose of Pseudocode

- Clarifies logic before coding  
- Bridges the gap between concept and code  
- Helps with problem solving  
- Makes logic easy to translate into any programming language  

---

## Why is Pseudocode Important?

- Reduces coding errors  
- Improves planning and structure  
- Makes debugging easier  
- Helps beginners understand programming logic  
- Saves time during implementation  

---

## Basic Structure of Pseudocode

- Start / End  
- Input statements  
- Process steps  
- Decision making (IF-ELSE)  
- Loops (if required)  
- Output statements  

Example structure:

```
START
INPUT values
PROCESS steps
IF condition THEN
    perform action
ELSE
    perform alternative action
END IF
OUTPUT result
END
```

---

## How to Translate Flowcharts into Pseudocode

- Start/End → START / END  
- Input symbol → INPUT  
- Process symbol → Assignment or calculation  
- Decision symbol → IF-ELSE  
- Arrows → Sequence of steps  

---

## Example of Pseudocode

### Problem1: Check whether a number is even or odd

```
START
INPUT number
IF number % 2 == 0 THEN
    PRINT "Even"
ELSE
    PRINT "Odd"
END IF
END
```  

### Problem2: Calculate the area of rectangle

```
START
INPUT length, width
SET area = length * width
PRINT area
END
```
---

## Best Practices for Writing Pseudocode

- Use simple and clear language  
- Keep steps logical and sequential  
- Avoid programming syntax  
- Use proper indentation  
- Keep it concise and readable  

---

## Real-World Applications of Pseudocode

- Software development planning  
- Algorithm design  
- Teaching programming concepts  
- System design discussions  
- Interview problem solving
